# Antigravity Quick Guide ⚡

**「思考停止で、最高品質を。」**
ここは日々の実務で使う **Action Terminal (操作パネル)** です。
やりたいこと（Scenario）を選び、コマンドブロックをそのままコピーしてチャット欄に貼り付けてください。

---

## ⚡ Quick Action (Copy & Paste)

### 🆕 新しい機能を作りたい
アイデアや要望を形にするための「要件定義」からスタートします。

```markdown
@[/spec-phase]
# 新機能の要望
作りたい機能: [ここに作りたいものを一言で]
目的/背景: [なぜこれが必要か]
---
プロのPMとして、必要な要件・仕様・機能一覧を全て提案してください。
```

### 🔨 バグを直したい・修正したい
既存のコードを安全に修正します。

```markdown
@[/impl]
# 修正依頼
対象: [バグの内容や修正したいファイル]
現状: [どうなっているか]
期待値: [どうなってほしいか]
```

### 🧹 コードを綺麗にしたい (Refactor)
機能は変えずに、可読性や保守性を上げます。

```markdown
@[/refactor]
対象: [リファクタリングしたいファイルパス]
指示: 可読性を上げ、ロジックを分離してください。（機能は変更しないでください）
```

### 🆘 迷ったら (AI Concierge)
何をしていいかわからない時は、コンシェルジュに聞いてください。

```markdown
@[/help]
現状のプロジェクト状況から、次に私が打つべきコマンドを1つだけ教えて。
```

---

## 🛡️ Security & Config

### 設定変更 (Security Level)
プロジェクトの規模に合わせてセキュリティ強度を変更します。

```markdown
@[/config]
現在のセキュリティプロファイルを確認し、必要であれば変更してください。
(L1: Startup / L2: Standard / L3: Enterprise / L4: National)
```

### 最終出荷監査 (Ship Gate)
リリースの準備ができたら、全テストと厳格な監査を行います。

```markdown
@[/ship]
実装が完了しました。
セキュリティ、品質、テストの全監査を実行し、リリース可能か判定してください。
```
