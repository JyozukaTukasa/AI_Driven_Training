## 1. 冒険の心得 (The Developer's Mindset)

この研修のゴールは、単にコードを書くことではなく、**「AIを活用して問題解決できるエンジニア」** になることです。
そのために、以下の3つのマインドセットを持ってください。

### 🧠 1. 暗記しない (The Drawer Strategy)
プログラミングの構文を完璧に覚える必要はありません。
**「引き出し（Index）」** を作ることが重要です。
> 「確か、**形の魔法 (HTML)** で箱を作る呪文があったな...（詳細はAIに聞こう）」
> これで十分です。詳細な記法はAIが補完してくれます。

### 🕵️ 2. まず仮説を立てる (Hypothesis First)
エラーが出た時、思考停止でAIにコピペしてはいけません。
**「たぶん、ここが怪しい」** と1行書いてからAIに聞く癖をつけてください。
- **Bad**: (無言でエラーを貼る)
- **Good**: 「CSSの色を変えた直後だから、スペルミスかもしれない。チェックして」
この「仮説」が、あなたのエンジニアとしての勘を育てます。

### 📢 3. 15分ルール & 報連相 (Communication)
- **15分ルール**: 15分自力（仮説検証）で解決できなければ、即座にAIまたはチームに相談する。
- **Slack日報**: エンジニアの仕事は「コードを書くこと」と「状況を伝えること」です。毎日必ず日報を書いてください。

## 2. システム全体像 (The Architecture)

![Web Three Roles](assets/0-2.png)

Webアプリの仕組みは、よく **「レストラン」や「銀行の窓口」** に例えられます。
この3つの役割（ロール）が連携して動いています。

### 🏰 Client (フロントエンド)
- **役割**: **「総合受付（窓口）」**
- **魔法のイメージ**: **形の魔法 (HTML)** と **色の魔法 (CSS)** で作られた「見た目」。
- **現実の例**: YouTubeの「動画プレイヤー」や「コメント欄」が表示されている、スマホの画面そのもの。

### 📡 Server (バックエンド/API)
- **役割**: **「実務担当（職員）」**
- **魔法のイメージ**: **呼び出し魔法 (API)** を使って、裏側で働く「見えないプログラム」。
- **現実の例**: 検索ボタンを押した瞬間に、裏側で動画を探してくるシステム。

### 🗄️ Database (データベース)
- **役割**: **「巨大書庫（アーカイブ）」**
- **魔法のイメージ**: **記憶の魔法 (Database)** で守られた「無限のノート」。
- **現実の例**: 世界中の何億本もの動画データが、消えずに保管されている場所。

### 🔄 データフロー (The Flow)
「AI（大賢者）」を使って、この3人を連携させるのがあなたの仕事です。

1.  **Request**: ユーザーが窓口（スマホ画面）で「猫の動画が見たい」と検索する。
2.  **Process**: 職員（Server）が依頼を受け取り、書庫（DB）へ走る。
3.  **Data**: 書庫から「猫動画リスト」を取り出す。
4.  **Response**: 職員が窓口に戻り、結果をユーザーに見せる。

### 🗺️ 今回の旅路
1.  **Day 1-2**: **Client** 大陸で、自分の城（サイト）を作る。
2.  **Day 3**: **Server** 島へ渡り、注文を聞くウェイター（API）を雇う。
3.  **Day 4**: **Database** (Cloud) に接続し、データを金庫にしまう。
4.  **Day 5**: 全てをつなげて、一つの巨大な「予約システム」を作る。
