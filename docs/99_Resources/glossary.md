# Technical Glossary (冒険の書・用語集)

エンジニアが使う専門用語を、身近な言葉に翻訳した辞書です。
「この単語なんだっけ？」と思ったら、ここに戻ってきてください。

> **🆘 エラーで困ったら？**
> 用語ではなくエラー解決策を知りたい場合は、**[Troubleshooting Guide (トラブルシューティング)](./troubleshooting.md)** を参照してください。

## 📅 Day 0: 旅支度 (Environment)

| 用語 (Term) | 直感的なイメージ | 身近な例 (Real World) |
|:---|:---|:---|
| **Cursor** | **魔法の杖** | あなたの代わりにコードを書いてくれる、AI搭載エディタ。 |
| **Git** | **セーブ機能** | ゲームのセーブデータ。失敗しても過去に戻れる。 |
| **Repository** | **作品の保管箱** | スマホの写真フォルダ（バックアップ対象）。 |
| **Clone** | **複製** | クラウドにあるデータを、自分の手元（ローカル）にコピーすること。 |
| **Commit** | **記録（確定）** | 書類にハンコを押すこと。「ここまでOK！」という印。 |
| **Push** | **転送** | Instagramに写真を投稿すること。手元のデータをクラウドへ送る。 |
| **Pull Request**| **レビュー依頼** | 「この変更を取り込んでください」と管理者に頼むこと。 |
| **Merge** | **合体** | 分かれていた作業（ブランチ）を本体（main）に統合すること。 |

## 📅 Day 1: 構造と見た目 (HTML/CSS)

| 用語 (Term) | 直感的なイメージ | 身近な例 (Real World) |
|:---|:---|:---|
| **HTML** | **骨組み（構造）** | 人体の「骨」。あるいは書類の「見出し・本文」。 |
| **Tag** | **命令札** | `<button>` (ボタンになれ), `<h1>` (大見出しになれ) という命令。 |
| **CSS** | **服・化粧（装飾）** | 人体の「服」や「メイク」。骨組みを可愛くする。 |
| **Property** | **特徴の指定** | `color: red` (色は赤), `width: 100px` (幅は100) などの指定。 |
| **Flexbox** | **整列の魔法** | 箱を「横並び」「真ん中寄せ」にするための最強の呪文。 |
| **Box Model** | **箱の構造** | すべての要素は「Margin(外)」「Border(枠)」「Padding(内)」を持つ箱である。 |

## 📅 Day 2: 動きの魔法 (JavaScript)

| 用語 (Term) | 直感的なイメージ | 身近な例 (Real World) |
|:---|:---|:---|
| **JavaScript** | **動き・魂** | 止まっている絵に命を吹き込むこと。「クリックしたら動く」など。 |
| **DOM** | **操作ハンドル** | JSからHTMLを操るための取っ手。`document.getElementById` など。 |
| **Event** | **きっかけ** | 「ボタンが押された」「マウスが乗った」などの出来事。 |
| **Listener** | **待ち伏せ** | 「チャイムが鳴ったら、ドアを開ける」のように、イベントを監視する人。 |
| **Console** | **裏話・ログ** | ユーザーには見えない、開発者専用のメッセージ表示画面 (F12)。 |
| **Variable (変数)** | **名前付きの箱** | `score = 10` のように、数字や文字を入れておく箱。 |

## 📅 Day 3: 指令所 (Backend API)

| 用語 (Term) | 直感的なイメージ | 身近な例 (Real World) |
|:---|:---|:---|
| **API** | **窓口・係員** | 厨房（DB）にオーダーを通してくれるウェイターさん。 |
| **Request** | **注文** | 「ハンバーグください」と頼むこと。 |
| **Response** | **返却** | 「はい、ハンバーグです」と料理を持ってくること。 |
| **Endpoint** | **窓口の場所** | 役所の「住民票係」「納税係」のような、特定の仕事をする場所（URL）。 |
| **JSON** | **荷姿・ダンボール** | データを送る時の専用の箱（フォーマット）。`{Key: Value}` の形。 |
| **Parameter** | **細かい注文** | `?size=medium` のように、注文にオプションをつけること。 |
| **Status Code** | **結果の合図** | `200` (OK!), `404` (ないよ), `500` (ミスった) などの番号。 |
| **CRUD** | **4つの基本操作** | Create(作), Read(読), Update(更), Delete(消)。データの扱いの基本。 |

| **Security Rules**| **門番** | 「IDを持ってる人しか入れない」などの守りのルール。 |

## 📅 Day 6+: 実務と納品 (Professional)

お客様と仕事をする上で、避けては通れない「エンジニアの共通言語」です。

| 用語 (Term) | 直感的なイメージ | 身近な例 (Real World) |
|:---|:---|:---|
| **Domain** | **住所（表札）** | `google.com` のような、人間が覚えられるWeb上の住所。 |
| **DNS** | **電話帳** | 「住所(Domain)」を「場所(IPアドレス)」に変換するシステム。設定しないと繋がらない。 |
| **SSL / HTTPS**| **暗号化（鍵）** | URLの横の鍵マーク。これがないサイトは「危険」とみなされ、客先で怒られる。 |
| **Cache** | **一時保存（残像）** | 「直したはずなのに直ってない！」という時は、大抵ブラウザが古い記憶（キャッシュ）を見ている。 |
| **Bug Report** | **犯行現場の証拠** | 「動きません」はNG。「どういう手順で、何が起きたか」を動画やスクショで伝える技術。 |
| **MVP** | **試作品** | Minimum Viable Product。全部作らず、「まずはここだけ」とお客さんと握るための最重要概念。 |

## 🛡️ Security & Web Mechanics (仕組みと防御)

「なぜログインできるの？」「なぜ危険なの？」を説明するための言葉です。

| 用語 (Term) | 直感的なイメージ | 身近な例 (Real World) |
|:---|:---|:---|
| **Cookie** | **会員証** | サーバーから渡される「私は会員です」という小さな手形。ブラウザが勝手に持ち歩いてくれる。 |
| **Session** | **滞在時間** | 「最後に行動してから30分」など、サーバーが覚えている有効期限。 |
| **LocalStorage** | **ブラウザのメモ帳** | Cookieと違い、サーバーには送られない私的なメモ。「ダークモード設定」などを保存する。 |
| **AuthN (認証)** | **本人確認** | 「あなたは誰？」(ID/Pass) を確認すること。Authentication。 |
| **AuthZ (認可)** | **権限確認** | 「入っていい？」(Admin/User) を確認すること。Authorization。 |
| **Vulnerability** | **セキュリティホール** | 防壁の「穴」。ここを突かれるとデータが盗まれる（脆弱性）。 |
| **Firewall** | **関所** | 変な通信を通さないためのガードマン。 |
| **WAF** | **Webの盾** | Webサイトへの攻撃（SQLインジェクションなど）を特化して防ぐ盾。 |

## 🏗️ Modern Architecture (現代の建築様式)

「どうやって作るか」のトレンド用語です。

| 用語 (Term) | 直感的なイメージ | 身近な例 (Real World) |
|:---|:---|:---|
| **SPA** | **一枚の画用紙** | ページ遷移せず、紙芝居のように中身だけ書き換えるアプリ (Reactなど)。サクサク動く。 |
| **SSR** | **事前描画** | サーバー側で絵を完成させてから送る方式 (Next.js)。Google検索に強い。 |
| **Microservices**| **分業制** | 巨大なアプリを「注文担当」「決済担当」などに分割して作る手法。 |
| **Serverless** | **場所借り** | サーバーを持たず、プログラムだけ置いて実行する方式 (Firebase Functionsなど)。 |

## 🤖 DevOps & Quality (自動化と品質)

「楽をするため」の技術群です。

| 用語 (Term) | 直感的なイメージ | 身近な例 (Real World) |
|:---|:---|:---|
| **CI/CD** | **全自動ロボット** | コードを書いたら自動でテストし、本番環境に公開してくれる仕組み。 |
| **Linter** | **掃除ロボット** | コードの汚れ（書式ミス）を自動で直してくれるツール。 |
| **Unit Test** | **部品テスト** | ネジ一本、関数一つが正しく動くか確認するテスト。 |
| **E2E Test** | **通しリハ** | ユーザーのフリをして、最初から最後までアプリを動かすテスト。 |
