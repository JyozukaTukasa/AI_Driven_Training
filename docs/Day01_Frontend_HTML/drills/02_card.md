# Drill 2: The Card (分子)

「ボタン」ができたら、次はそれを組み合わせた「カード（Card）」を作ります。
Instagramの投稿や、Amazonの商品、YouTubeの動画サムネイル…これらは全部「カード」という構造です。
**「縦に積む」** という感覚を掴みましょう。

## 🎯 Goal (完成イメージ)
以下の要素が入った「カード」を画面の真ん中に作ってください。

-   **画像**: 上部に大きく表示
-   **タイトル**: 画像の下に太字で「絶景スポット」
-   **説明文**: タイトルの下に「素晴らしい景色です。」
-   **ボタン**: 一番下に「詳細を見る」

---

## Step 1: Block Sketch (ブロック分解)
「親箱」の中に「子箱」がどう入っているか、言葉でスケッチします。

**【構造メモ】**
-   カード全体（親箱）
    -   縦並び（Flex Column）
    -   全体に影がついている
-   中身（子箱たち）
    1.  画像（img）
    2.  タイトル（太文字）
    3.  説明文（普通の文字）
    4.  ボタン（詳細を見る）

---

## Step 2: Visual Dictionary (辞書引き)
[Visual Dictionary](../../99_Resources/visual_dictionary.md) を見て確認しましょう。

-   **「縦に積む」** とはどう言えばいい？
    -   → （辞書を見る）「縦積みにして」「縦に並べて」
-   **「画像」** は？
    -   → `<img>` (画像)

---

## Step 3: Construct (構築)
AIに「やりたいこと」を伝えます。

### 🤖 Prompt Example
```text
【作りたいもの】
「観光ガイド」に出てくるような、写真付きのカードを作りたいです。

【構造】
1. カードの箱 (class: card)
   - 幅はスマホくらいのサイズ (300px)
   - 「縦積み」に並べてください。
   - 角を丸くして、影をつけて、カードっぽくしてください。

2. 中身
   - 上：画像 (picsum.photos/300/200 というURLを使ってください)
   - 中：タイトル「絶景スポット」 (文字大きめ)
   - 中：説明文「素晴らしい景色です。」
   - 下：ボタン「詳細を見る」 (幅いっぱいに)

【配置】
- このカードを、画面のど真ん中に置いてください。
```

---

## Step 4: Code Literacy (解剖)
生成されたコード（HTML）を見て、以下の「名詞」を見つけてください。

1.  **画像の場所（URL）はどこ？**
    -   `picsum.photos...` という文字を探してください。
    -   それは `src="..."` という場所（属性）に書かれていますか？
2.  **「絶景スポット」という文字はどこ？**
    -   `<h2>` や `<h3>`、または `<b>` で囲まれていますか？
3.  **「実験」: 画像を変えてみる**
    -   `picsum.photos/300/200` の `200` を `300` に書き換えてみてください。
    -   画面の画像が「正方形」になりましたか？

> **Why?**: これがわかれば、WordPressやReactで「画像が古い！」と言われた時、**「どこを直せばいいか」**が瞬時にわかるようになります。
