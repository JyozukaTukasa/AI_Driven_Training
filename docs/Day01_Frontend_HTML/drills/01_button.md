# Drill 1: The Button (原子)

Webサイト構築の最小単位である「ボタン」を作ります。
たかがボタンですが、ここには **「形」「色」「動き（ホバー）」** というCSSの基礎が詰まっています。

## 🎯 Goal (完成イメージ)
以下の特徴を持つボタンを画面の真ん中に表示してください。

-   色: 鮮やかな青 (`#3b82f6`)
-   形: 角が少し丸い
-   影: ふわっと浮いている
-   動き: マウスを乗せると、少し色が濃くなる

---

## Step 1: Block Sketch (ブロック分解)
まず、日本語で構造を分解します。
（今回は単純ですが、このプロセスを省略しないでください）

**【構造メモ】**
-   画面全体（親箱）
    -   真ん中に配置したい
    -   背景は白でいい
-   ボタン（子箱）
    -   青い色
    -   白い文字「Click Me」
    -   角丸
    -   影付き

---

## Step 2: Translation (翻訳)
[Visual Dictionary](../99_Resources/visual_dictionary.md) を見て、日本語をタグに翻訳しましょう。

| 日本語 | 翻訳 (Tag / CSS) |
|:---|:---|
| 画面全体 | `<body>` または `<div class="container">` |
| 真ん中に配置 | `display: flex; justify-content: center; align-items: center;` |
| ボタン | `<button>` |
| 青い色 | `background-color: #3b82f6;` |
| 角丸 | `border-radius: 8px;` |
| 影付き | `box-shadow: ...` |
| マウスを乗せる | `:hover` |

---

## Step 3: Construct (構築)
翻訳した言葉を使って、AIに「やりたいこと」を伝えます。
無理にコードを書く必要はありません。**「何が」「どこに」「どうなってほしいか」** を言葉にしてください。

### 🤖 Prompt Example
「タグ名」や「クラス名」を無理に使う必要はありません。
「どんな箱」で「どうなっているか」を、以下のように伝えてみましょう。

```text
【作りたいもの】
画面の真ん中に、青いボタンを1つ置きたいです。

【構造】
1. ベースの箱
   - 画面いっぱいの大きさ
   - 中身を「上下左右の真ん中」に揃えてください。

2. ボタン
   - テキストは "Click Me"
   - 「角を丸く」してください。
   - 「影」をつけて、ふわっと浮かせてください。
   - 色は「青」で、マウスを乗せたら「濃く」なるようにしてください。
```

> **Point**: AIは「真ん中に揃えて」「丸くして」といった**「やりたいこと（機能・見た目）」**を伝えれば理解してくれます。
> 難しい言葉は使わなくてOKです！

---

## Step 4: Code Literacy (解剖)
**ここが最重要ステップです。**
AIが書いたコードは「魔法」ではありません。あなたの言葉が翻訳されたものです。
生成されたコード（HTML/CSS）を見て、以下のクイズに答えてください。

1.  **「ボタン」の本体はどこ？**
    -   `<button ...>` というタグを見つけてください。
2.  **「色」を決めているのはどこ？**
    -   `background-color: ...` という行を探してください。
3.  **「実験」: 色を変えてみる**
    -   その `blue` や `#3b82f6` を `red` に書き換えてみてください。画面のボタンが赤くなりましたか？

> **Why?**: この「コードと見た目の対応関係」がわかれば、**言語がPythonやSwiftに変わっても「色を変える場所」を見つけられるようになります。**
> これが「どの環境でも戦えるエンジニア」の第一歩です。

---

## Step 5: Variations (応用実験)
**ここからが本番です。**
AIに頼らず（またはAIと相談しながら）、以下の改造を行ってください。

1.  **形を変える**: 丸ボタン (`border-radius: 50%`) にしてみてください。
2.  **動作を変える**: マウスを乗せたら「大きくなる (`transform: scale`)」ようにできますか？
3.  **状態を変える**: クリックしたら「押された！」という文字に変わるようにできますか？（これはJSが必要かも...ChatGPTに聞いてみよう）

> **Self-Study**: 全部のドリルで、この「3つの改造」を自分で考えて実行してください。
> 「言われた通りにやる」のは作業です。「自分で変えてみる」のが学習です。
