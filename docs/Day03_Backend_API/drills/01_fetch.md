# Drill 1: The Fetch (データ取得)

Day 3 からは **バックエンド（城の裏側）** です。
自分のパソコンの中だけでなく、インターネットの向こう側からデータを持ってきます。

## 🎯 Goal
-   ボタンを押すと、**「東京の天気」** が表示される。
-   (今回は日本気象庁などの公開API、またはシンプルなダミーAPIを使います)

---

## Step 1: Block Sketch
1.  **イベント**: ボタンを押す。
2.  **通信**: インターネットに「東京の天気教えて」と投げる。
3.  **待機**: 答えが返ってくるまで待つ (Async/Await)。
4.  **表示**: 返ってきた答えを画面に出す。

---

## Step 2: Visual Dictionary
-   **「通信する」** → `🌍 fetch(url)`
-   **「待つ」** → `⏳ await`
-   **「データ形式」** → `JSON`

---

## Step 3: Construct

### 🤖 Prompt Example
```text
【作りたいもの】
天気予報ボタン。

【構造】
- ボタン「天気を取得」
- 結果表示エリア (div)

【動き】
ボタンが押されたら、以下の外部APIからデータを取ってきてください (Fetch)。
URL: https://api.open-meteo.com/v1/forecast?latitude=35.68&longitude=139.76&current_weather=true

返ってきたデータの中から `temperature` (気温) を取り出して、画面に表示してください。
```

---

## Step 4: Code Literacy
1.  **URLはどこ？**
    -   `https://...` と書かれている場所を探してください。
    -   ここを `latitude=43.06` (札幌) に変えると、気温はどうなりますか？
2.  **「待て」の合図は？**
    -   `await fetch(...)` の `await` が「待て」の命令です。これがないと、データが届く前に次の行に進んでしまい、エラーになります。
