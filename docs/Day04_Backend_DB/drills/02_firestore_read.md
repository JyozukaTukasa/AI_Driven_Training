# Drill 2: The Projector (データを映す)

保存したデータは、読み出せなければ意味がありません。
クラウドにある「手紙」を全部取ってきて、画面に並べます。

## 🎯 Goal
-   ページを開くと、Firestoreに保存されているメッセージが一覧表示される。

---

## Step 1: Block Sketch
1.  **イベント**: ページが開いた時 (または「読み込み」ボタン)。
2.  **通信**: クラウドからデータを全部もらう (`getDocs`)。
3.  **表示 (ループ)**: もらった手紙を一枚ずつ、画面に追加していく。

---

## Step 2: Visual Dictionary
-   **「全部とってくる」** → `getDocs(collection)`
-   **「一枚ずつ処理」** → `.forEach((doc) => { ... })`

---

## Step 3: Construct

### 🤖 Prompt Example
```text
【作りたいもの】
Firestoreのデータを一覧表示するリスト。

【構造】
- 表示エリア (`<ul id="list">`)
- 更新ボタン (`<button>`)

【動き】
1. 更新ボタンを押したら、Firestoreの `messages` コレクションからデータを全件取得してください (`getDocs`)。
2. 取得したデータ（複数）を順番に処理して (`forEach`)、
   それぞれの `text` をリスト項目 (`<li>`) として画面に追加してください。
```

---

## Step 4: Code Literacy
1.  **「繰り返し」の場所**
    -   `forEach` という単語を探してください。
    -   これが「データが100個あれば100回繰り返す」命令です。
2.  **「中身を取り出す」場所**
    -   `doc.data().text` のような記述はありますか？
    -   ここが「封筒から手紙を取り出す」瞬間です。

> **Point**: 「配列 (List)」と「繰り返し (Loop)」はセットで使われます。
> 「データがいくつあるかわからない」ときは、必ずこのパターンを使います。
