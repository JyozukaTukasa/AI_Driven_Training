# Mission: 予約システム "Starry Night"

Day 5、これまでの全てをぶつける総合演習です。
あなたは、人気グランピング施設「Starry Night」から予約システムの開発を依頼されました。

## 🎯 Goal
**設計から実装、デプロイまでを独力で行う。**

1.  **Design**: `design.md` を書き、チャットボットにレビューしてもらう (Drillの内容)。
2.  **Develop**:
    -   Frontend: React (Day 1-2の知識)
    -   Backend: Firebase (Day 3-4の知識)
3.  **Deploy**: 実際にWeb上に公開し、スマホで予約できるようにする。

---

## Rule: The 15-Minute Rule
Day 5 では、**「詰まったら15分だけ悩んでいい」** です。
それでもわからなければ、**「何に詰まっているか」** を言語化して、メンター（AI）に相談してください。
（答えを聞くのではなく、ヒントをもらうこと）

## Process
1.  **Setup**: 新しいフォルダ `starry-night` を作り、プロジェクトを開始。
2.  **Design**: `design.md` を作成 (Drill参照)。
3.  **Frontend**: 「見た目」から作る (HTML/CSS)。
4.  **Backend**: 「データ」をつなぐ (Firestore)。
5.  **Refine**: 「予約がいっぱいです」などのエラー表示を作る。

**Good Luck, Engineer.**
これが完成すれば、あなたはもう「初心者」ではありません。
「AIを使って課題を解決できるエンジニア」です。

---

## Step 6: Hardening (最終試験)
動くだけでは納品できません。以下の「意地悪なテスト」をパスしてください。
バグが見つかったら、AIに修正依頼を出して直してください。

### 🧪 QA Checklist
1.  **連打テスト**:
    -   「予約ボタン」を0.1秒に10回連打してください。
    -   予約が10件重複して作られませんか？（ローディング中はボタンを無効化できていますか？）
2.  **過去日予約**:
    -   「昨日」の日付で予約できてしまいませんか？
3.  **巨大データ**:
    -   名前に「あ」を1000文字入力したらどうなりますか？（レイアウト崩れやエラー）

> **Reflection**: 
> AIは「作れ」と言われたものは作りますが、「守れ」と言われなければ守りません。
> **「何を守るべきか」を知っていること**。それが、あなたがAIを使う理由です。
