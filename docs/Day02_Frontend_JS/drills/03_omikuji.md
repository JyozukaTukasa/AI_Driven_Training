# Drill 3: The Omikuji (おみくじ)

最後は「ランダム（偶然）」と「リスト（配列）」です。
ガチャ、おすすめ表示、広告…「たくさんの中から1つ選ぶ」機能です。

## 🎯 Goal (完成イメージ)
-   「運勢を占う」ボタンがある。
-   押すと、「大吉」「中吉」「凶」のどれかがランダムに表示される。

---

## Step 1: Block Sketch (ブロック分解)
1.  **データ (リスト)**: くじの中身（大吉、中吉、凶...）を用意する。
2.  **ロジック (抽選)**: サイコロを振って、リストから1つ選ぶ。
3.  **表示**: 画面の文字を書き換える。

---

## Step 2: Visual Dictionary (辞書引き)
-   **「リスト」** → `📦 データ (const results = ["大吉", "凶"...])`
    -   ※ 辞書には載っていませんが、`[ ]` で囲むのがリストの印です。

---

## Step 3: Construct (構築)

### 🤖 Prompt Example
```text
【作りたいもの】
ボタンを押すと運勢が出る「おみくじアプリ」を作りたいです。

【構造 (HTML)】
- 結果表示エリア (`<div id="result">`): 最初は「???」
- 占いボタン (`<button>`)

【動き (JavaScript)】
ボタンが押されたら、以下の手順で動きます。

1. くじのリストを用意する (配列)
   - ["大吉", "中吉", "小吉", "凶", "大凶"]
2. ランダムに1つ選ぶ
   - `Math.random()` を使って、リストの中から1つ選んでください。
3. 結果を表示する
   - 選ばれた文字を、画面の「???」の場所に表示してください。
   - 大吉なら文字を赤く、凶なら青くしてください。
```

---

## Step 4: Code Literacy (解剖)
1.  **「くじの中身」はどこ？**
    -   `["大吉", "中吉"...]` と書かれた行を探してください。
    -   ここに **「超大吉」** を書き足してみましょう。出るまでボタンを連打してください！
2.  **「サイコロ」はどこ？**
    -   `Math.random()` という呪文を探してください。
    -   これがコンピュータの中でサイコロを振っている場所です。

> **Challenge**: 「大吉」が出る確率を増やせますか？（ヒント：リストの中に「大吉」を3つくらい書けば…？）
