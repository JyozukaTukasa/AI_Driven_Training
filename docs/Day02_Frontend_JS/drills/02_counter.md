# Drill 2: The Counter (カウンター)

次は「数字」を扱います。
「いいねボタン」や「買い物かごの数」など、Webアプリは数字の管理（計算）の連続です。

## 🎯 Goal (完成イメージ)
-   画面に「0」という数字と、「+」「-」の2つのボタンがある。
-   「+」を押すと数字が増える。
-   「-」を押すと数字が減る。
-   **ルール**: 0より小さく（マイナス）にはならない。

---

## Step 1: Block Sketch (ブロック分解)
1.  **データ (変数)**: 今の数字を覚えておく箱が必要（最初は0）。
2.  **イベント**: 「+」が押された時、「-」が押された時。
3.  **ロジック**:
    -   +の時: 箱の中身に 1 を足す。
    -   -の時: 箱の中身から 1 を引く（ただし、0より大きい時だけ）。
4.  **表示**: 画面の数字を書き換える。

---

## Step 2: Visual Dictionary (辞書引き)
-   **「数字の箱」** → `📦 データ (let count = 0)`
-   **「もし0より大きければ」** → `⚖️ もし〜なら (if)`

---

## Step 3: Construct (構築)

### 🤖 Prompt Example
```text
【作りたいもの】
シンプルなカウンターアプリを作りたいです。

【構造 (HTML)】
1. 数字を表示する場所 (`<h1>`): 最初は "0"
2. ボタンエリア:
   - 「+」ボタン (`<button>`)
   - 「-」ボタン (`<button>`)

【動き (JavaScript)】
1. 「現在の数」を覚える変数を用意してください (初期値0)。
2. 「+」が押されたら、変数を +1 して、画面の数字を更新してください。
3. 「-」が押されたら、変数を -1 して更新してください。
   - ただし、**0より小さくならないように (if)** してください。
```

---

## Step 4: Code Literacy (解剖)
1.  **「0」の箱はどこ？**
    -   `let count = 0;` (または `score` など) という行を探してください。
    -   この `0` を `100` に書き換えて、リロードするとどうなりますか？（初期値が変わるはずです）
2.  **ブレーキ（if文）はどこ？**
    -   `if (count > 0)` のような記述を探してください。
    -   これが「マイナスに行かせない」ための守り神です。

> **Point**: アプリ開発とは、「変数（箱）」の中身を増やしたり減らしたりすることだと言えます。
